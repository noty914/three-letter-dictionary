# 用語候補（アルファベット別）

非エンジニアが業務の場で意味が曖昧なまま進みそうな観点のメモです。ひな形の用語ファイル（`api` / `http` / `dns`）とは別の洗い出し用です。

---

## A

- **API** … 「つなぐ」「公開する」と一言で済まされがちで、画面の裏で何が起きているかまで想像しにくい。
- **AWS** … クラウドの代表名として会議に出るが、何がサービス名で何が一般名詞かが混線しやすい。

## B

- **BCP** … 災害・障害時の事業継続で必ず出るが、ITの話なのか総務・経営の話なのか境界が曖昧になりやすい。

## C

- **CRM** … 顧客管理の話で頻出だが、SFAやMAと何が違うかで止まりやすい。
- **CSV** … 表データの受け渡しで使うが、文字コードや区切りで壊れる経験とセットで初めて怖さがわかる。
- **CLI** … 「黒い画面で打つ操作」と聞くが、GUIとの違いや、なぜエンジニア作業に寄りやすいかが腹落ちしにくい。
- **CPU** … Central Processing Unit。一般的には「計算の頭脳」と説明されがちだが、コア数・クロック・スマホの SoC（SoC 内の CPU 部分）など話が広がると、**何が速さのボトルネックか**が腹落ちしにくい。

## E

- **ERP** … 基幹システムの総称として出るが、導入範囲と業務変革の話が一体化して重く感じられやすい。

## F

- **FAQ** … 問い合わせ削減の文脈で出るが、更新されず形骸化しているとユーザー不信につながる点が見えにくい。

## G

- **GCP** … クラウドの選択肢としてAWSと並ぶが、契約主体や課金の見え方が社内調整でつまずきやすい。
- **GPU** … Graphics Processing Unit。画像・動画だけでなく、生成AIの学習・推論の文脈でも頻出。**CPU との役割分担**や、クラウド課金・オンプレ増設の話まで来ると止まりやすい。
- **GUI** … 画面操作の話で出るが、CUIやAPIとの対比がないと意味が薄まる。
- **GWS** … Google Workspace（旧 G Suite）。Gmail や Drive などのセット名として出るが、契約範囲やデータの置き場・主権の話まで来ると境界が曖昧になりやすい。

## I

- **IDE** … 開発環境として紹介されるが、エディタとの違いが曖昧になりやすい。
- **IoT** … センサー連携の話で頻出だが、セキュリティや保守の話まで含まれると急に重い。

## J

- （このアルファベットはベースリストに未記載。追記用）

## K

- **KPI** … 指標の話で最頻出。数値の定義が曖昧なまま運用されると現場不信につながりやすい。
- **KGI** … 経営目標として経営会議で出るが、KPIとの上下関係や粒度が組織ごとにブレると現場が動きにくい。
- **KSF** … 成功要因として戦略の文脈で出るが、KPI・施策に落とす前の「要因」として抽象的なまま終わりやすい。

## L

- **LAN** … 社内ネットの話で出るが、Wi‑FiやVPNとの包含関係で図が頭に描けないと混乱しやすい。
- **LLM** … 生成AIの中核としてニュース・企画で急増。学習データや著作権の話まで一気に来て止まりやすい。

## M

- **MFA** … 多要素認証として導入方針に必ず出るが、利便性とのトレードオフで現場反発が起きやすい。
- **MVP** … 「最小限で出す」と訳されがちで、品質や責任範囲の話が抜け落ちやすい。

## N

- **NDA** … 秘密保持契約として法務とセットで出るが、技術情報の範囲が境界線になりやすい。

## O

- （このアルファベットはベースリストに未記載。追記用）

## P

- **PIN** … 暗証番号として日常語に近いが、端末ロックや決済の責任分界でトラブルになりやすい。
- **POC** … 概念実証として「まず動くもの」を求められがちだが、期間・成功条件・本番移行の線引きが曖昧だと終わりが見えにくい。

## Q

- **QCD** … 品質・コスト・納期（デリバリー）のトレードオフとして製造・プロジェクトで頻出。IT化やDXの文脈に入ると「どれを犠牲にするか」の説明が難しく感じられやすい。

## R

- **ROI** … 投資対効果として必ず出るが、分母分子の定義が曖昧だと数字が信じられなくなる。
- **RPA** … 業務自動化として導入検討で急増。例外処理や運用保守の話が後追いになりやすい。
- **RAG** … 生成AIが社内文書などを参照して答える仕組みとして出るが、参照範囲・更新頻度・権限の話がないと期待値が膨らみやすい。
- **RFP** … Request for Proposal（提案依頼書）。ベンダー選定で必ず出るが、評価基準・決裁・スコープが曖昧だと比較が公平に見えず揉めやすい。

## S

- **SQL** … データ抽出の話で出るが、Excel感覚で頼むと破綻しやすい（権限や負荷の話がついてくる）。
- **SSO** … シングルサインオンとして「一度ログインすれば複数システムに入れる」話で出るが、対象範囲・退職時の権限外し・例外アカウントの運用で止まりやすい。

## T

- **TLS** … セキュア通信として「SSLの後継」として出るが、証明書更新や中間証明書でトラブルになりやすい。

## U

- **UAT** … 受入試験としてベンダー・社内の境界で出るが、合格条件が曖昧だと終わらない。

## V

- **VPN** … リモートワークで必須だが、「入れば安全」誤解や速度低下の説明でつまずきやすい。

## W

- **WBS** … プロジェクトの作業分解で出るが、成果物ベースと作業ベースの混在で形が崩れやすい。
- **WWW** … 古いが文書に残る。Webサイト全体を指すのか曖昧なまま使われやすい。

## X

- **XML** … データ形式としてレガシー文脈で出るが、JSONとの比較がないと価値が伝わりにくい。

## メモ

- ベースにあった `#追加！…#` は、上記の **CLI / GWS / KGI / KSF / QCD** として本文に組み込みました。
- **GWS** は略称として会話に乗りやすい一方、正式名称は **Google Workspace** です（辞典では英語併記を推奨）。
- **QCD** は 3 語の頭文字の組み合わせ略語で、辞典の「3文字1語」ルールとどう扱うかだけ決めると運用しやすいです。
- **RPF と RFP**: 調達・ベンダー文脈では **RFP**（Request for Proposal）が標準表記。**RPF** は誤記のことが多いです。別領域で RPF が定義されている場合は、その業界用語として別項目にするとよいです。
- **WBS** は、今回のご指定以前から本ファイルの **W** に収載済みです。

## ほか足しやすい候補（アルファベット未整理・会話で効くもの）

次に辞典や本リストへ足すときのプール例です。

- **SLA** … 可用性・障害対応など「契約上の約束」として会議に必ず出る。
- **SLO**／**SLI** … SLA より細かい目標・指標の話でセットで出る。
- **PRD**／**BRD** … 何を作るか／なぜ作るかのレイヤー違いで止まりやすい。
- **RFC**／**ADR** … 方針や設計判断の記録としてリンクだけ飛んでくる。
- **OAuth**／**OIDC**／**SAML** … 認証方式の違い。SSO とセットで出る。
- **RBAC** … ロールベースの権限。誰が何をできるかの設計で頻出。
- **DLP** … 情報漏えい対策。稟議・セキュリティ方針で出る。
- **ETL** … 分析前のデータ加工。止まったときの担当範囲で揉めやすい。
- **CI**／**CD** … 自動テスト・自動リリース。納期や品質の説明で出る。
- **ステージング**／**本番** … どこで再現・検証するかの線引き。
- **Webhook** … 外部連携で「届かない」トラブル時に名前が出る。
- **キャッシュ** … 更新が反映されない理由説明で超頻出。
- **トークン**（AI）… 文字数・コストの説明で出る。
- **ハルシネーション** … 生成AIの誤答リスクの話。
- **インシデント**／**ポストモーテム** … 障害後の振り返りと再発防止。
