## 初級

「ソフト同士が話すための窓口・約束事」と捉えるとよいです。アプリやサービスが、他のシステムに「このデータをください」「この処理をお願いします」と頼むときの入口が API です。

補足（比喩）：レストランでいうと「メニュー（取れるものの一覧）」と「厨房へのオーダー方法」に近いです。

## 中級

多くの場合、HTTP 上で URL・メソッド・パラメータ・レスポンス形式が決まっており、JSON などでデータをやり取りします。公開 API を設計・公開する側と、それを呼び出すクライアント側に役割が分かれます。

## 上級

インターフェース仕様（スキーマ、認可、バージョニング、レート制限、後方互換）が設計の中心になります。REST、GraphQL、gRPC などは「API をどう切るか」のスタイルの違いです。
