## 初級

Web のページや API のやり取りの土台となる「約束事」です。ブラウザやアプリがサーバーに「この情報をください」と頼み、サーバーが応答する流れで使われます。

補足（比喩）：郵便の「宛名・種類・返信ルール」に相当するイメージです。

## 中級

リクエストとレスポンスがペアになり、メソッド（GET / POST など）、URL、ヘッダー、ステータスコード（200 / 404 など）で意味が整理されます。HTTPS は HTTP に暗号化を足したものです。

## 上級

HTTP/2・HTTP/3 など多重化や伝送の最適化、キャッシュ制御（Cache-Control）、認証（Authorization ヘッダーと組み合わせることが多い）が論点になります。
